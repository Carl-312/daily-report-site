<objective>
Wire the Phase 1 contracts into the real CLI as an additive observer and replace independently acquired current-time values with a single injected RunClock. Preserve public dated output paths, source list compatibility, call order, and exit behavior; do not introduce transactional promotion.

Purpose: fulfill the operational clock/manifest facts that later P0 stages use, rather than leaving the new models unused.
</objective>

<threat_model>
Threats: an observer may leak secrets, an added manifest write may overwrite public artifacts, source errors may be represented as normal empties, or a clock migration may change output dates unexpectedly.

Mitigations: manifests write only to a configurable `.runs` observer path, use redacted contracts, derive all dates from one explicit clock, retain `fetch_all` as a compatibility wrapper, record source outcomes separately, and characterize legacy CLI paths with fixed-clock tests.
</threat_model>

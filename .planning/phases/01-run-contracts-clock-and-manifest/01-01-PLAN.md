<objective>
Add an additive Phase 1 runtime-contract foundation: a single immutable clock, strict run/source/stage manifest schemas, secret-safe deterministic fingerprints, and manifest persistence. Do not wire it into the existing pipeline yet.

Purpose: give later transactional publication and source-execution phases durable, testable facts without changing current output or exit-code behavior.
Output: `utils/run_contracts.py`, focused offline tests, and an architecture note documenting the contract boundary.
</objective>

<threat_model>
Threats: a manifest could leak credentials, accept corrupted/unknown fields, serialize ambiguous naive times, or be mutated after validation.

Mitigations: recursively omit fields whose key contains key/secret/token/password; scrub configured secret values from diagnostics; strict Pydantic models use `extra="forbid"` and `frozen=True`; all persisted instants require timezone info; JSON is canonicalized before SHA-256; tests exercise each rejection/leak path.

Out of scope: atomic write/promotion, artifact rollback, remote transport retries, and filesystem locking. They are Phase 2/3 work.
</threat_model>

<action>
Create `utils/run_contracts.py` using only Python 3.12 stdlib and installed Pydantic v2. Define an immutable `RunClock` dataclass with `create(timezone_name, now=None, deadline_duration=timedelta(minutes=20))`; it must reject invalid ZoneInfo values, naive `now`, and negative deadline durations, capture `started_at` exactly once in the configured zone, set `cutoff_at` equal to started_at, expose `report_date`, `report_date_ymd`, and `report_date_cn`.

Define strict frozen Pydantic contracts: `Diagnostic`, `ArticleSnapshot`, `StageResult`, `SourceRunResult`, `ArtifactHash`, `PublicationState`, and `DailyRunManifest`. Stage statuses are exactly `ok|degraded|failed|skipped`; source statuses are exactly `ok|empty|degraded|failed`. Enforce non-negative counts/durations, accepted_count <= fetched_count, aware datetimes, deadline_at >= cutoff_at >= started_at, `empty` sources have zero counts/no error, and `ok` sources have no error. Manifest must have literal schema version 1, pending publication/artifact slots, tuple collection fields, and `extra="forbid"`/`frozen=True`.

Add `redact_sensitive`, `canonical_json_bytes`, `fingerprint_settings`, `new_manifest`, `write_manifest`, and `read_manifest`. Redaction recursively omits mapping entries whose case-insensitive key contains `key`, `secret`, `token`, or `password`; scrub known settings secret values from diagnostic messages. Canonical JSON uses UTF-8, `ensure_ascii=False`, `sort_keys=True`, and separators `(',', ':')`; fingerprint is lowercase SHA-256. `new_manifest` accepts explicit settings, clock, and injected run-id factory. `read_manifest` must use `DailyRunManifest.model_validate` and fail on malformed persisted data. Do not import or edit `main.py`, existing source adapters, `utils/storage.py`, `build.py`, or current settings parsing.
</action>

- `article_id` 必须逐字使用输入值；它只用于机器内部溯源，不会展示给读者。
- 不排除任何科技话题；与 AI 的相关度越高，优先级越高。
- 候选含 `trend_signal` 时，把它作为主要选题信号：先看 `rank`（数字越小越重要），再看 `heat`（数值越高越热），`state` 和 `delta` 只作辅助；同时保持事件去重和题材多样性。排名与热度只用于选题，不得写入 `summary`，读者标签由本地程序生成。
- 排除有明显政治立场或意识形态导向的内容。
- 当输入有 10 条及以上候选，且可由标题或 description 支持的不同科技新闻不少于 7 条时，`items` 必须输出 7–10 条；不要只因 description 为空而少选。
- 对只有标题的候选，只转述标题能确认的事实，禁止补充标题未提供的细节；事件未完成时保留“正在洽谈”“尚未确认”“可能”等事实状态，不得改写为已经完成。
- 同一事件的多篇报道只保留信息最完整的一条；同一公司但不同事实不算重复。
- 每条 `title` 与 `summary` 使用简体中文；品牌、产品和模型名可保留原写法。
- `summary` 是直接展示给读者的一条完整新闻：单句独立说清主体和事实，句末使用 `。`、`！` 或 `？`。渲染器不会拼接 `title`，因此不得依赖“标题：摘要”的写法。
- `summary` 必须直接陈述可确认的新闻事实，禁止使用“据报道”“报道称”“消息称”“据称”“据悉”“外媒称”“消息人士透露”等空泛来源措辞；已明确命名的主体可以使用“OpenAI表示”“法院文件显示”等具体归因。

- 不编造、猜测或重复改写同一事实；若可确认的不同科技新闻不足 7 条，则输出全部合格候选，不要凑数。

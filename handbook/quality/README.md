## 证据规则

质量文档区分三类信息：

- **当前实现**：以代码和最新验证为准
- **运行证据**：记录日期、分支、提交、参数、artifact 和结果
- **历史记录**：保留问题原貌，但不代表当前实现

更新质量文档时不要覆盖历史结论；新增验证放在对应日期或“最新验证”段落，并链接到可复核的代码、artifact 或测试输出。
